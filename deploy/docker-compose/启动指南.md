# DjangoBlog Docker部署启动指南

## 准备工作

1. **确保Docker Desktop已安装并运行**
   - 在Windows系统中，请点击开始菜单，找到并启动Docker Desktop应用程序
   - 等待Docker Desktop完全启动（状态栏显示绿色）

2. **检查Docker服务是否正常运行**
   - 打开命令提示符或PowerShell
   - 运行 `docker --version` 确认Docker已安装
   - 运行 `docker info` 确认Docker服务正在运行

## 启动项目

1. **进入docker-compose目录**
   ```
   cd c:\Users\20725\Desktop\blog\DjangoBlog\deploy\docker-compose
   ```

2. **构建并启动容器**
   ```
   docker-compose up -d --build
   ```

3. **检查容器运行状态**
   ```
   docker-compose ps
   ```

## 访问项目

服务启动后，您可以通过以下方式访问项目：
- Web访问: http://localhost:8888
- Django管理后台: http://localhost:8888/admin

## 常用命令

- **查看日志**: `docker-compose logs -f`
- **停止服务**: `docker-compose down`
- **重新构建**: `docker-compose build`

## 注意事项

1. 如果遇到端口冲突，可以修改docker-compose.yml文件中的端口映射
2. 首次启动时会自动创建数据库和初始数据
3. 确保您的系统满足Docker Desktop的最低要求

- 使用阿里云Docker镜像源: registry.cn-hangzhou.aliyuncs.com/library/
- 使用阿里云Python包镜像源: https://mirrors.aliyun.com/pypi/simple/
